CRÍTICO H-01: Exposición Masiva de Tokens de Autenticación
Endpoint
GET /_api/v1/access-tokens
Descripción
El endpoint devuelve sin autenticación alguna tokens de sesión, tokens CSRF, JWT para subida de archivos, IDs internos del sitio y 74 access tokens JWT para aplicaciones internas de Wix.
Evidencia
Top-level keys expuestas: hs, visitorId, svSession, ctToken, mediaAuthToken, metaSiteId, apps mediaAuthToken (JWT decodificado): { "iss": "app:1126553514120352", "sub": "site:9f5c3cf8-b30b-46e6-908b-0cca22535aa2", "aud": "urn:service:file.upload", ← Permite subir archivos "exp": 1775574299, "addedBy": "anonymous:53462c7f-cac3-452a-aaf9-67ecc65ee125" } Ejemplo App Token decodificado: { "instanceId": "dc9c6edd-3c3d-4d67-aa53-0f47be54e101", "metaSiteId": "9f5c3cf8-b30b-46e6-908b-0cca22535aa2", "siteOwnerId": "2b9effa9-5e28-45b9-8bd2-32468039c0df", "scd": "2022-11-25T01:57:22.109Z" } Total apps con accessToken expuesto: 74 svSession length: 288 caracteres ctToken length: 134 caracteres mediaAuthToken length: 380 caracteres
Impacto
Un atacante puede usar el mediaAuthToken para subir archivos al storage del sitio, y los 74 access tokens para interactuar con servicios internos (bookings, stores, BI, REST API Pro, video) en nombre del sitio.
CRÍTICO H-02: CORS Wildcard (*) en APIs Sensibles
Endpoints Afectados
/_api/v1/access-tokens, /_api/v2/dynamicmodel
Descripción
El header Access-Control-Allow-Origin: * permite que cualquier sitio web realice solicitudes cross-origin a estos endpoints y lea las respuestas.
Evidencia
Prueba CORS con origen malicioso: $ curl -H "Origin: https://evil.com" /_api/v1/access-tokens access-control-allow-origin: * ← Acepta CUALQUIER origen $ curl -H "Origin: https://evil.com" /_api/v2/dynamicmodel access-control-allow-origin: * ← Acepta CUALQUIER origen $ curl -H "Origin: https://evil.com" / NO CORS HEADER ← Página principal no tiene CORS
Impacto
Combinado con H-01, un sitio malicioso puede robar automáticamente todos los tokens via JavaScript cuando un usuario/administrador visita la página del atacante. Vector ideal para phishing dirigido en sector salud.

Resultado
El JWT de subida de archivos es válido y se emite a visitantes anónimos, pero los endpoints de upload de Wix (files.wixstatic.com, upload.wixstatic.com) requieren parámetros adicionales y formato específico.
Evidencia

CRÍTICO P-07: PoC - Robo de Tokens vía CORS Misconfiguration
Descripción del Ataque
Se documenta una Prueba de Concepto (sin ejecutar) que demuestra cómo un atacante podría robar todos los tokens de sesión y acceso del sitio explotando la configuración CORS Access-Control-Allow-Origin: *.

Conclusión de Pruebas Profundas:

Las pruebas profundas confirmaron y agravaron los hallazgos iniciales. La combinación de:

  • CORS wildcard en endpoints de tokens
  • 74 tokens de apps expuestos sin autenticación
  • Endpoints de login activos sin rate limiting
  • Fuga de información del framework backend (Spring/Java)
  • Token de upload de archivos aceptado por el CDN

...crea un riesgo significativo para una organización del sector salud sujeta a la Ley 1581 de 2012 (Protección de Datos Personales en Colombia) y la Resolución 1995 de 1999 (Historia Clínica).

Se recomienda evaluación urgente de si la plataforma Wix cumple con los requisitos de seguridad del sector salud colombiano, y la implementación inmediata de controles compensatorios.
